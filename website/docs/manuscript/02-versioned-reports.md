# Phase 1 — Versioned reports

Snapshot the analysis state at every milestone into `data/publications/v<x>/`
so the manuscript can reference stable, reproducible figures and numbers.

## The concept

A **versioned report** is a self-contained Quarto document that:

- Pins a specific commit / DVC version of the data
- Renders to PDF + HTML
- Generates all figures + tables it cites
- Lives at `data/publications/v<n>/<topic>.qmd`
- Has a git tag matching its version (`v0.1-eda`, `v0.5-baseline`, `v1.0-final`)

The manuscript pulls **figures and numbers from a specific report version**,
not from `data/08_reporting/` directly. Decoupling means the manuscript can
quote v0.5 numbers in round-1 review and v0.7 numbers in round-2 review
without rewriting prose.

## Plant `data:publications`

The `data:publications` component gives you the landing directory:

```bash
just grow data:publications
```

```
data/publications/
├── v0.1/        ← initial EDA snapshot
├── v0.5/        ← post-feature-engineering + baseline
├── v0.7/        ← after reviewer round-1 changes
└── v1.0/        ← final preprint version
    ├── eda-report.qmd
    ├── eda-report.pdf            ← generated
    ├── eda-report.html           ← generated
    ├── classifier-report.qmd
    ├── classifier-report.pdf
    ├── figures/
    │   ├── pairplot.png
    │   ├── boxplots.png
    │   └── confusion-matrix.png
    └── tables/
        └── classification-metrics.csv
```

## Workflow: penguins study, snapshot v0.1

After the EDA step in the [penguins tutorial](/docs/penguins/explore), you
have a notebook and figures in `data/08_reporting/`. Snapshot them as
**v0.1-eda**:

### 1. Create the report directory

```bash
mkdir -p data/publications/v0.1/{figures,tables}
```

### 2. Copy the figures used by the manuscript at this milestone

```bash
cp data/08_reporting/082_figures/penguins_pairplot.png \
   data/publications/v0.1/figures/pairplot.png
cp data/08_reporting/082_figures/penguins_boxplots.png \
   data/publications/v0.1/figures/boxplots.png
```

### 3. Create the report Quarto

```bash
cat > data/publications/v0.1/eda-report.qmd <<'EOF'
---
title: "Penguins study — exploratory data analysis (v0.1)"
author: Your Name
date: "2026-05-15"
abstract: |
  Snapshot of pre-modeling exploratory analysis. Used as the basis for
  manuscript Section 2 (Methods → Data exploration).
format:
  html: default
  pdf: default
---

# Sample distribution

After dropping rows with missing key fields, n = 333 records remain:

| Species   | n   |
|-----------|-----|
| Adelie    | 146 |
| Gentoo    | 119 |
| Chinstrap |  68 |

# Pairwise feature scatter

![Pairplot of measurements by species](figures/pairplot.png){#fig-pairplot}

# Per-species distributions

![Box plots of measurements](figures/boxplots.png){#fig-boxplots}

# Notes for manuscript

- Gentoo separable on flipper_length and body_mass
- Adelie / Chinstrap require bill_length to disambiguate
- Bill ratio (length/depth) is a strong domain feature

# Provenance

- Source data: `data/01_raw/011_external/penguins.csv` @ commit `a1b2c3d`
- Generated by: `notebooks/02-exploration/eda/<ts>_eda_penguins-overview.qmd`
EOF
```

### 4. Render the report

```bash
quarto render data/publications/v0.1/eda-report.qmd
```

Outputs `eda-report.html` and `eda-report.pdf` next to the source.

### 5. Tag the version

```bash
git add data/publications/v0.1/
git commit -m "v0.1-eda: snapshot EDA + pairplot + boxplots"
git tag -a v0.1-eda -m "Penguins manuscript draft 0.1: EDA snapshot"
```

If you have DVC active, also `dvc add data/publications/v0.1/` to track the
binary outputs.

## Workflow: snapshot v0.5 after model is trained

After [Phase 5 of the penguins tutorial](/docs/penguins/model), you have the
random forest classifier and a confusion matrix.

```bash
mkdir -p data/publications/v0.5/{figures,tables}

# Carry forward v0.1 figures (still relevant)
cp data/publications/v0.1/figures/{pairplot,boxplots}.png \
   data/publications/v0.5/figures/

# Add new model-stage figures
cp data/08_reporting/082_figures/cm_rf.png \
   data/publications/v0.5/figures/confusion-matrix.png

# Snapshot evaluation metrics as a stable CSV
cp data/model-output/073_evaluation/test_report.txt \
   data/publications/v0.5/tables/classification-report.txt
```

Create `data/publications/v0.5/classifier-report.qmd`:

```markdown
---
title: "Penguins study — random forest classifier (v0.5)"
author: Your Name
date: "2026-05-22"
abstract: |
  Baseline + candidate classifier results. Used for manuscript
  Section 4 (Results) and Figure 3 (Confusion matrix).
format:
  html: default
  pdf: default
---

# Method recap

- Train: 213 / val: 53 / test: 67 (stratified, random_state=42)
- Features: 4 raw + 2 engineered (`bill_ratio`, `body_density_proxy`)
- Models: multinomial logreg (baseline) + random forest n=200 (candidate)

# Results

Test-set accuracy: random forest = 0.99, logreg = 0.97

![Confusion matrix on test set (Random Forest)](
figures/confusion-matrix.png){#fig-cm}

@fig-cm shows perfect Gentoo classification; one Chinstrap with unusually
short bill misclassified as Adelie.

# Provenance

- Models: `data/models/062_candidate/rf.joblib` @ commit `b3c4d5e`
- DVC pipeline stage: `train` (run timestamp 2026-05-22T10:30Z)
- MLflow run: `runs:/abc123def456`
```

Render + tag:

```bash
quarto render data/publications/v0.5/classifier-report.qmd
git add data/publications/v0.5/
git commit -m "v0.5-baseline: classifier results snapshot"
git tag -a v0.5-baseline -m "Penguins manuscript draft 0.5: baseline classifier"
```

## How the manuscript uses versioned reports

In `writeup/manuscript/manuscript.qmd`:

```markdown
# Methods — Data exploration

After dropping rows with missing key fields, **n = 333** records remained
(see @fig-pairplot for cluster structure).

![Pairplot of measurements by species (v0.5)](
../../data/publications/v0.5/figures/pairplot.png){#fig-pairplot}

# Results

The random forest classifier achieved 0.99 test accuracy
(see @fig-cm for the confusion matrix).

![Confusion matrix (Random Forest, n_estimators=200)](
../../data/publications/v0.5/figures/confusion-matrix.png){#fig-cm}
```

The manuscript references **v0.5 specifically**. If the underlying analysis
changes (you re-run with new hyperparameters, or someone fixes a bug), v0.5
remains immutable. The next manuscript revision pins to v0.7 or v1.0.

## Versioning cadence

A pragmatic cadence:

| Tag | When | What's in `data/publications/v<x>/` |
|---|---|---|
| `v0.1-eda` | After exploratory analysis | Pairplots, distributions, summary stats |
| `v0.3-features` | After feature engineering | + Domain features, scaled features tables |
| `v0.5-baseline` | After first model | + Baseline + candidate model results |
| `v0.7-final-model` | After hyperparameter tuning | + Production model + cross-validation |
| `v1.0-preprint` | Frozen for preprint submission | Full set, locked |
| `v1.1-revision` | After reviewer round 1 | + Updated figures from reviewer requests |
| `v1.2-revision` | After reviewer round 2 | + Final accepted version's figures |

## Automating snapshots

Add a recipe to `tasks/manuscript.just`:

```just
# Snapshot current analysis state as data/publications/v<version>/
snapshot version label="snapshot":
    #!/usr/bin/env bash
    set -euo pipefail
    dest="data/publications/v{{version}}"
    mkdir -p "$dest/figures" "$dest/tables"
    cp data/08_reporting/082_figures/*.png "$dest/figures/" 2>/dev/null || true
    cp data/08_reporting/081_tables/*.csv  "$dest/tables/"  2>/dev/null || true
    cp data/model-output/073_evaluation/*.txt "$dest/tables/" 2>/dev/null || true
    git add "$dest"
    echo "✓ Snapshot at $dest. Now write $dest/<topic>-report.qmd, then:"
    echo "    quarto render $dest/<topic>-report.qmd"
    echo "    git commit -m 'v{{version}}-{{label}}: snapshot'"
    echo "    git tag v{{version}}-{{label}}"
```

```bash
just snapshot 0.5 baseline
just snapshot 1.0 preprint
```

## Verifying provenance

To re-render an old report from its frozen sources:

```bash
git checkout v0.5-baseline
dvc checkout                                                 # if DVC-tracked
quarto render data/publications/v0.5/classifier-report.qmd
git switch -                                                 # back to main
```

The output will be byte-identical (modulo render timestamps) to the original
v0.5 PDF — provenance is preserved.
